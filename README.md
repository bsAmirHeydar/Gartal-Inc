# Gartal-Inc

**MQL4/MQL5 trading automation and backtesting workflow project focused on order logic, position sizing, risk management, and backtest data handling.**

Gartal-Inc is a trading automation project developed around MetaTrader workflows. The repository includes MQL4/MQL5 source files and utility modules for strategy execution, backtest-speed experimentation, backtest file management, and saving test results.

The project demonstrates practical experience with trading-system development, automation, risk logic, and the operational side of backtesting workflows.

---

## Project Purpose

The purpose of this repository is to build and test reusable components for trading automation and backtesting workflows.

The project focuses on:

- MQL4/MQL5 strategy execution
- Entry and exit automation
- Position sizing
- Capital and risk management
- Backtest-speed experimentation
- File-based backtest result storage
- Workflow utilities for strategy testing

---

## Main Files

| File / Folder | Purpose |
|---|---|
| `main.mq4` | Main MQL4 trading program / strategy entry point. |
| `backtest speed.mq4` | Backtest-speed experimentation script / EA. |
| `save backtest.mq4` | Tool for saving or exporting backtest-related information. |
| `backtest-file-manager.mqh` | Backtest file handling and management utilities. |
| `MQHs/` | Reusable MQL header modules. |
| `.vscode/` | Editor configuration. |
| `status.txt` | Project status / runtime notes. |
| `temp.txt` | Temporary project file. |

## How to Use

1. Copy `.mq4` files and the `MQHs/` folder into the appropriate MetaTrader 4 project folders.
2. Open the files in MetaEditor.
3. Compile the main EA/script files.
4. Run the tools in MetaTrader Strategy Tester or a controlled demo environment.
5. Review generated outputs and logs.
